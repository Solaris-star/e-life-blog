## 一、核心问题

GPT-image-2 逐帧生成**无法保证角色一致性**——每帧的角色外貌、比例、姿态都会漂移，不适合做帧序列。像素动画帧必须由同一个工具/流程一次性产出，保持绝对一致。

**结论**：像素动画帧序列交给 Claude Code 制作，GPT-image-2 只适合出概念参考图。

### 小章鱼形态（32×32）

| 特征 | 规范 |

## 四、需要制作的帧序列

### 4.1 人形 idle 呼吸动画（64×64）

- 帧数：8-12 帧
- FPS：8
- 循环：是
- 动作描述：
  - 帧1-4：身体微下沉（呼吸吸气），触手轻微外摆
  - 帧3：**眨眼**（闭眼帧）
  - 帧5-8：身体回升（呼吸呼气），触手回摆
  - 帧7：**眨眼**（闭眼帧）
  - 末帧回到帧1姿态，无缝循环
- 关键：所有帧角色外貌绝对一致，仅位移/触手角度/眨眼差异

### 4.2 人形 → 章鱼变身过渡（64×64 → 32×32）

- 帧数：6-8 帧
- FPS：12（快速变身感）
- 循环：否（播放后切入章鱼 idle）
- 动作描述：
  - 帧1：人形正常站姿（= idle 帧1）
  - 帧2-3：身体发出蓝光，触手开始增多变大，身体缩小
  - 帧4-5：人形轮廓模糊，章鱼轮廓浮现，罗盘冠不变
  - 帧6-7：完全变为章鱼形态
  - 帧8：= 章鱼 idle 帧1

### 4.3 章鱼 → 人形变身过渡（32×32 → 64×64）

- 帧数：6-8 帧
- FPS：12
- 循环：否
- 动作：逆序，章鱼发光变大 → 触手减少 → 人形轮廓浮现 → 完全人形

### 4.4 小章鱼 idle 漂浮动画（32×32）

- 帧数：4-6 帧
- FPS：6
- 循环：是
- 动作描述：
  - 帧1：基准漂浮，触手下垂
  - 帧2：上浮 1-2px，触手微展
  - 帧3：下沉 1px，触手微收
  - 帧4：回到基准
  - 可穿插1帧眨眼

### 4.5 小章鱼 hover 反应（32×32）

- 帧数：2-3 帧
- FPS：10
- 循环：否（播放后回到 idle）
- 动作描述：
  - 帧1：触手迅速收缩蜷起
  - 帧2：身体微微后缩
  - 帧3：回到正常（接 idle）

### 4.6 文末 Solaris 坐姿读书（64×64）

- 帧数：单帧静态图
- 动作：坐在小桌前，手持书卷/羽毛笔，桌上有墨水瓶和茶杯
- 最多加 CSS 呼吸微动（`transform: scaleY` 交替）

## 五、形态切换交互逻辑

- **自动轮播**：每 30 秒自动切换形态（人形 idle → 变身 → 章鱼 idle → 变身 → 人形 idle）
- **点击切换**：点击角色立即触发变身（打断当前轮播计时器，重新开始 30s 倒计时）
- **变身不可打断**：变身帧序列播放期间不接受第二次点击

## 六、技术方案

Canvas 2D + `requestAnimationFrame` + Sprite Sheet 帧序列

```typescript
const ANIMATIONS = {
  humanIdle:     { frames: [0..7],   fps: 8,  loop: true  },
  humanToOcto:   { frames: [8..14],  fps: 12, loop: false },
  octopusIdle:   { frames: [15..20], fps: 6,  loop: true  },
  octopusToHuman:{ frames: [21..27], fps: 12, loop: false },
  octopusHover:  { frames: [28..30], fps: 10, loop: false },
};
```

渲染方式：`drawImage` 从 sprite sheet 截取对应帧区域，64×64 和 32×32 可各一份 sprite sheet。
