# Solaris 数字花园 — 全站交互叙事设计

> 状态:草案 v2 · 待 Solaris 审定
> 设计标准:歧路旅人 HD-2D —— 像素角色小而精致,场景环境为视觉主体
> 最后更新 2026-06-17

---

## 0. 核心理念

**整个网站是一座数字花园,Solaris 是住在里面的看守者(外神)。**

用户浏览网站 = 在花园里走动。每个页面是花园的一个区域,Solaris 在不同区域做不同的事。页面切换时角色"走"过去,形成连贯的探索叙事。

不是"每个页面贴一个动画",是"一个角色活在整座花园里"。

### 歧路旅人比例标准

- 角色高度 ≈ 场景可视高度的 **1/4 ~ 1/3**(不是占满)
- 场景环境(建筑/植物/家具/地形)是视觉主体
- 角色小但动作细腻 —— 行走/待机/转身/交互都有独立帧
- 透视:2.5D 俯视斜角,角色 sprite 站在"地面线"上

## 2. 主线故事(探索叙事)

**主题:一个访客来到数字花园,Solaris 从暗中观察 → 出现引导 → 最后送别。**

不是循环播放,是**随用户浏览路径展开**的单向叙事(用户首次访问时):

| 章节 | 触发条件 | Solaris 在做什么 | 思考飘字 |
|------|----------|------------------|----------|
| **序章·察觉** | 首次进首页 | 从花园入口的树后探出半个身子,冷眼观察 | 「……又有人闯进来了。」 |
| **第一章·打量** | 滚动到首页下方 | 走出来站在小径旁,手背在身后 | 「技术、生活、还有没说出口的念头——都埋在这座花园里。」 |
| **第二章·引路** | 首次点进文章页 | 站在书架旁,手指向某本书 | 「随便逛。能读懂多少,看你。」 |
| **第三章·陪伴** | 在文章页停留 >30s | 搬了小凳子坐在旁边看书 | 「……我在看这本。你呢。」 |
| **第四章·好奇** | 进 Daily 页 | 趴在溪边茶亭栏杆上看水 | 「今天的天气,有点意思。」 |
| **第五章·放松** | 进想法页 | 躺在草坡上看天,触手散开 | 「碎碎念也不错。不必每句都有意义。」 |
| **第六章·整理** | 进资源页 | 在储物木屋里翻箱倒柜 | 「攒了不少东西。有些还能用。」 |
| **终章·送别** | 回到首页(已浏览过其他页) | 站在花园门口,微鞠躬 | 「慢慢看吧。反正我,等了很久了。」 |
| **彩蛋·变身** | 点击角色 3 次 | 变身章鱼,触手乱舞 | 「……别戳。这副样子,比较好藏。」 |

**已浏览过的页面角色会记住** —— 第二次进同一个页面,角色动作会变化(比如文章页从"指书"变成"坐着看你来")。

## 3. 各页面角色设计

### 3.1 首页 / (花园入口)

**场景**:花园铁艺拱门 + 爬藤 + 远处树影
**角色位置**:拱门右侧,约占场景高度 1/4
**动作序列**:
- 序章:从树后探头(半身) → 缩回 → 再探头(完整站立)
- 待机:站在拱门旁,缓慢呼吸,偶尔转头看访客方向
- 引路:手指向花园深处(页面下方)
- 送别:站到门口中央,微鞠躬

**交互**:
- 悬停角色 → 眼神追随鼠标
- 点击角色 → 说一句反应词(「……别戳。」)
- 滚动到第二屏 → 角色从入口走到小径旁

### 3.2 文章列表 /articles (花园小径)

**场景**:石板小径 + 两侧书架(像图书馆走廊)
**角色位置**:小径中段,书架旁
**动作**:
- 首次到访:站在书架前,手指向某本书
- 停留:搬凳子坐下,翻开自己的书
- 再次到访:抬头看你一眼,继续低头看书

**交互**:
- 悬停文章卡片 → 角色转头看向你悬停的那本书
- 点击文章 → 角色站起来目送你进去

### 3.3 文章详情 /articles/[slug] (阅读亭)

**场景**:小型阅读亭,木质,有窗
**角色位置**:亭子门槛上坐着
**动作**:
- 待机:坐着晃腿,翻书页
- 你停留 >30s:她合上书,歪头看你
- 滚动到文末:她站起来,指向评论区方向

### 3.4 Daily /daily (溪边茶亭)

**场景**:日式茶亭 + 溪流 + 樱花树
**角色位置**:茶亭栏杆旁趴着
**动作**:
- 待机:趴在栏杆上看水面
- 偶尔:伸手捞水里的落叶
- 有新 Daily:抬头看天,指指云

### 3.5 想法 /ideas (草坡)

**场景**:缓草坡 + 远山 + 浮云
**角色位置**:草坡顶部躺着
**动作**:
- 待机:躺着,触手(章鱼形态)或发丝散开随风飘
- 偶尔:翻身,换一边躺
- 悬停某条想法:坐起来看那个方向

### 3.6 资源 /resources (储物木屋)

**场景**:木屋内部 + 满墙置物架 + 箱子
**角色位置**:木屋中央,蹲着翻箱子
**动作**:
- 待机:蹲着翻东西,偶尔拿起物品端详
- 悬停资源:站起来,把物品递向你
- 找到好东西:举起物品晃两下

### 3.7 写作专题 /writing (山顶书阁)

**场景**:高山上的玻璃书阁,360° 窗景
**角色位置**:书阁中央,站在梯子上
**动作**:
- 待机:站在梯子上整理高处的书
- 偶尔:下来走一圈检查
- 悬停专题:从梯子上指那个方向

### 3.8 归档 /archive (地下书库)

**场景**:地下圆顶书库,螺旋楼梯
**角色位置**:楼梯中段坐着
**动作**:
- 待机:坐着翻旧册子
- 偶尔:打哈欠(这里太安静了)
- 悬停年份:翻到对应那页

## 4. 交互系统

### 4.1 全局交互

| 触发 | 反应 |
|------|------|
| 悬停角色 | 眼神/头部追随鼠标 |
| 点击角色 | 反应词(轮换池) + 轻微弹跳 |
| 连续点击 3 次 | 强制变身章鱼(彩蛋) |
| 久无操作 20s | 打盹,「Zzz……」飘字 |
| 任何操作 | 惊醒,回到当前章节状态 |
| 页面切换 | 角色走出画面 → 新页面角色走入 |

### 4.2 页面专属交互

每个页面的交互见 §3 各小节。核心是**角色的动作和页面的内容联动** —— 不是自顾自循环,是"看着你在做什么"。

### 4.3 章鱼形态(彩蛋)

- 连续点击 3 次触发,持续 10s 后自动变回
- 章鱼形态下交互不同:
  - 悬停 → 触手伸向鼠标
  - 点击 → 墨汁四溅特效
  - 移动 → 章鱼漂浮跟随一段

## 5. 技术架构

### 5.1 角色尺寸标准(歧路旅人比例)

```
场景高度 H(可视区域)
角色高度 = H × 0.22 ~ 0.28
角色宽度 = 角色高度 × 0.5(苗条体型)
cell 尺寸 = 角色高度 × 1.2(留白 + 阴影)
```

示例:Hero 区 400px 高 → 角色 ~100px 高 → cell ~120px(比现在的 168px 小很多)

### 5.2 性能优化(解决卡顿)

1. **缩小 canvas** — 不再 520×520,按场景实际需要设定(首屏约 320×200)
2. **DPR cap 降到 1.5** — 像素风不需要 2x 超采样
3. **素材瘦身** — 变身 sheet 从 565KB 压到 < 150KB
4. **离屏暂停** — IntersectionObserver 已有,确保 viewport 外不渲染
5. **按需加载** — 只加载当前页面的 spritesheet,不预载全部
6. **帧率限制** — 非 morph 动画 8fps 足够,rAF 内节流

### 5.3 状态管理

```typescript
// 全局叙事状态(Context 或 Zustand)
interface GardenState {
  visitedPages: Set<string>;      // 已访问的页面
  currentChapter: Chapter;         // 当前章节
  characterForm: 'human' | 'octo'; // 当前形态
  lastInteraction: number;         // 上次交互时间
  storyProgress: number;           // 0-7 章节进度
}
```

### 5.4 素材清单(需重新生成)

| 素材 | 帧数 | 说明 |
|------|------|------|
| human_idle | 6 | 站立呼吸(缩小版) |
| human_walk | 8 | 行走循环(4方向×8帧 或 2方向翻转) |
| human_turn | 4 | 转身 |
| human_sit | 1 | 坐着(文章页) |
| human_lean | 1 | 趴着(Daily页) |
| human_lie | 1 | 躺着(想法页) |
| human_point | 2 | 指向(引路) |
| human_bow | 3 | 鞠躬(送别) |
| morph_to_octo | 16 | 变身(精简版) |
| morph_to_human | 16 | 变回 |
| octo_idle | 6 | 章鱼漂浮 |
| octo_reach | 2 | 触手伸出(交互) |

总素材量比现在多,但每个更小(120px cell vs 168px),总内存反而降低。

## 7. 设计决策(Solaris 已确认)

1. **故事基调**:A 档"冷淡带暖" ✓
2. **章鱼形态**:保留彩蛋(连续点击 3 次触发) ✓
3. **移动端**:<768px 显示静态定帧(不隐藏) ✓
4. **页面切换**:游戏原生式 —— 角色活在全局固定层,路由变化时角色不消失,切换 sprite + 位置"走"到新场景。用 Next.js View Transitions API 实现,不用独立组件淡入淡出。
5. **首屏**:自动播放动画,角色就是生活在这个网站里的 ✓

## 8. 实施优先级

### Phase 1:紧急修复(当前卡顿 + 比例)
- 缩小角色 cell 从 168px → 120px(歧路旅人比例)
- canvas 尺寸从 520×520 → 适配场景(首屏约 320×200)
- DPR cap 2 → 1.5
- 素材压缩(变身 sheet 565KB → <150KB)
- 移动端静态定帧(不隐藏)

### Phase 2:全局角色架构
- 角色组件提升到全局 layout 层
- View Transitions API 接入路由切换
- 叙事状态管理(visitedPages / currentChapter)

### Phase 3:多场景素材生成
- 用 gpt-image-2 生成各页面专属 spritesheet
- 行走/转身/坐/躺/趴/指/鞠躬 等动作帧
- 场景背景插画(各页面)

### Phase 4:逐页接入 + 交互打磨
- 8 个页面场景 + 角色站位
- 页面专属交互(悬停联动、点击反应)
- 主线章节触发逻辑
