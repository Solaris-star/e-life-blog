## 它到底是什么

![LightCDN 免费流量与节点选择](/images/agnes/lightcdn-free-500gb/lightcdn-01-pricing-network-v1.png)

LightCDN 是一个轻量 CDN 服务。

官网的定位很明确：做全球加速，尤其强调 **东南亚、中东、非洲、南美** 这些传统 CDN 容易变贵、节点覆盖不均的区域。

它的计费也比较简单：功能基本不单独收费，主要看流量。

目前公开价格里，按量付费分两档：

| 网络档位 | 价格 | 节点 | 适合谁 |
|---|---:|---:|---|
| Core Network | **$0.005 每 GB** | 8 个国际节点 | 个人站、小流量、先测试 |
| Standard Network | **$0.03 每 GB** | 32 个国际节点 | 更广覆盖、业务更稳定的站 |

所以它的核心卖点不是“免费到天荒地老”。

而是：**先给你 500GB 试用空间，然后超出后价格也不算离谱。**

## 三级域名也能加

![三级域名接入 LightCDN 的路径](/images/agnes/lightcdn-free-500gb/lightcdn-02-cname-flow-v1.png)

这个点我觉得很实用。

很多 CDN 或安全服务，对域名层级、根域验证、接入方式会卡得比较细。

LightCDN 的文档里，接入逻辑比较常规：

1. 在控制台创建 CDN Resource。
2. 填要加速的域名。
3. 填源站地址。
4. 选择 Core Network 或 Standard Network。
5. 控制台给你一个 CNAME 目标。
6. 去 DNS 服务商那里添加解析。

如果你是子域名，比如：

```text
cdn.example.com
img.example.com
api.example.com
```

那就直接用 **CNAME**。

如果你想加速根域名，比如：

```text
example.com
```

那要看 DNS 服务商支不支持 **ALIAS、ANAME 或 CNAME Flattening**。

因为标准 DNS 里，根域名一般不能直接放 CNAME，容易跟 MX、NS 等记录冲突。

所以最省心的方式还是：**优先用子域名接入。**

## 可以拿来做什么

![LightCDN 适合怎么用](/images/agnes/lightcdn-free-500gb/lightcdn-03-use-cases-v1.png)

我觉得它最适合这几类用途。

### 1. 小博客和静态站

比如图片、CSS、JS、附件下载。

500GB 对普通个人站来说很宽裕。

如果你的访问主要来自海外，或者东南亚、中东这类地区，它比一些国内免费 CDN 更值得试。

### 2. 图床和附件分发

对象存储、VPS 目录、静态文件服务都可以放后面。

前面套 LightCDN，用来缓存常见资源。

别拿大文件疯狂刷。

这种免费额度服务，最怕你把它当不限量下载站。

### 3. 反代入口

如果源站是一个海外 VPS，直连速度不稳定，可以试着套一层 CDN 看效果。

有用户反馈，线路状态好的时候，穿透代理可以跑到 4K 观看级别。

这个我会保留但不神化。

因为最终速度取决于三件事：

- 源站本身速度
- LightCDN 节点调度
- 你本地网络到节点的线路

CDN 不是魔法棒。

源站太慢，套什么都救不回来。

### 4. 备用线路

这个场景我反而最推荐。

主线路用你熟悉的服务。

LightCDN 做备用域名。

哪天主 CDN 抽风，DNS 切一下，至少还有第二条路。

## 使用时注意这几个坑

### 免费额度别当 SLA

免费额度随时可能调整。

官方公告里也出现过从 100GB 调整到 500GB 的迁移说明。

这说明规则是会变的。

所以，不要把“免费”写进自己的长期架构里。

### 先测节点，再迁移

CDN 的体验非常看地区。

你最好拿真实用户所在地区测一下：

- 首包时间
- 静态资源加载
- HTTPS 握手
- 回源稳定性
- 晚高峰表现

别只看一次测速截图。

网络这种东西，像猫。白天乖，晚上未必乖。😤

### 注意缓存规则

如果你反代的是动态服务，不要一股脑全缓存。

登录态、API、后台路径都要小心。

静态资源可以缓存久一点。

动态页面要按需设置。

### 根域名优先别碰

不是不能做。

是麻烦。

新手直接用：

```text
cdn.example.com
```

比折腾根域名舒服很多。
